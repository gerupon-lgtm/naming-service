### T-003: character_master 初期シードデータ投入
- 依存: T-002
- 完了条件: 常用漢字・ひらがな・カタカナの画数データが登録され、熊崎式の部首補正が反映されている
- 検証方法:
  - 正常系: 部首補正マッピング表（氵→+1画等）を作成し、Kanji alive APIから取得した部首情報と突き合わせて該当文字を自動検出・補正できる
  - 正常系: 補正後、代表的なサンプル文字（氵を含む字、扌を含む字等）で正しい熊崎式画数になっている
  - 境界: Kanji aliveの対応漢字（1235字）に含まれない文字は、MuzukanjiAPIで部首情報が取得できるか試す。取得できればそちらを使って自動補正、できなければ手動で確認しリストに反映する
  - 事前検証: MuzukanjiAPIは着手前に実際のレスポンス形式・無料枠の上限を一度確認し、使えるかどうかをまず判断する（RapidAPIのAPIキー登録が必要）
  - 注記: Kanji alive API・MuzukanjiAPIともにこのシードデータ生成スクリプトでのみ使用し、本番の実行時APIとしては呼び出さない

### T-005: 姓名診断ロジック（五格計算）
- 依存: T-002, T-003, T-004
- 完了条件: 入力された姓名から天格・人格・地格・外格・総格・総合スコア・ランクが計算される
- 検証方法:
  - 正常系: 既知のサンプル姓名で五格が期待値と一致
  - 異常系: character_masterにもkanjiapi.devにも存在しない文字を含む入力→`DIAGNOSIS_UNAVAILABLE`エラーを返す
  - 異常系: kanjiapi.devがタイムアウト・エラーを返す場合、フォールバックせずエラー表示のみ行う（フォールバックAPI不採用の方針通り）
  - 境界: 1文字の姓・名（極端に短い入力）でも計算が破綻しない

### T-007: S-001/S-002 画面実装
- 依存: T-005, T-006
- 完了条件: ユーザーが姓名を入力し診断結果画面まで到達できる
- 検証方法:
  - 正常系: 入力→実行→結果表示のE2E
  - 異常系: 未入力での実行操作が抑止される
  - 境界: 診断処理が10秒近くかかった場合もプログレス表示が継続し、タイムアウトエラーにならない

### T-008: URLパラメータ共有機能

  - 異常系: URLパラメータが不正・欠落している場合、S-001（入力画面）へ安全に誘導する

### T-013: LLMコメント生成基盤（Ollama→OpenRouterフォールバック）構築＋姓名診断結果への適用
- 依存: T-005（診断結果の構造化データが確定している必要がある）
- 完了条件: 診断結果を即時表示した後、LLMコメントが非同期で追加表示される
- 検証方法:
  - 正常系: Ollamaが正常応答→そのコメントが診断結果の下に表示される
  - 異常系: Ollamaがタイムアウト（5秒超過）またはエラー→OpenRouterにフォールバックしてコメント表示
  - 異常系: 両プロバイダとも応答不可→コメント領域が表示されず、診断結果本体の表示・操作（ファイル出力等）に影響しない
  - 設定: `LLM_FALLBACK_ORDER` を入れ替えた場合、指定順でプロバイダが呼ばれることを確認
  - 境界: LLM応答がロジック計算済みの数値・ランクと矛盾する内容を生成しても、画面上の数値・ランク表示（ロジック側の値）は上書きされない

### デプロイ・構成メモ（実装で確定した事項）
- ホスティングは当初想定（フロント=GitHub Pages／API=Vercel）を変更し、**全部Vercelにまとめる**構成に。`vercel.json`（framework:null, outputDirectory:frontend/dist）＋ルート `package.json` の build で対応。
- `/api` 配下の共有ロジックは `api/_lib`（先頭 `_` でVercelの関数化対象外）に配置。
- `/api` のTS関数は**CommonJSで統一**（`api/package.json` に `type:module` を置かない＋`api/tsconfig.json` は `module:CommonJS`/`moduleResolution:node`）。拡張子なしのimportを使うため。
- 環境変数（DATABASE_URL, LLM_OLLAMA_*, LLM_FALLBACK_ORDER 等）はVercelダッシュボードで設定。
- DBは Neon(Postgres)。未設定時は同梱seed辞書にフォールバックして動作。

## 現状サマリと残タスク（v1.18 時点）

**実装・公開済み**: Phase1（MVP: 姓名診断＋LLMコメント＋性別＋各格の成り立ち＋三才/五行）と、Phase2 の中核（F-002〜F-005 ペット命名提案・F-011 候補コメント・S-003/S-004 画面・タブ切替）まで動作。本番: naming-service-red.vercel.app（全部Vercel）。

**残タスク（デプロイ・運用）**
- 最新コード（ZIP）をGitHubへ反映 → Vercel自動デプロイ。
- **Neonの再投入**: 文字マスタが約3,168字（かな濁音・半濁音・拗音・促音・々を追加）に増えているため、`db/setup_all.sql` を再実行（UPSERTで更新）。
- LLMの環境変数調整（任意）: 生成・コメントで自然だったモデルを優先するなら `LLM_FALLBACK_ORDER` を調整。

**残タスク（機能・任意）**
- name_master のさらなる候補拡充（現在 約216件。DB化済み）。
- 四柱推命アプリとの五行連携の実配線（出力側 `wuxing` は準備済み・相手側I/F待ち）。
- 独自ドメイン設定。
- 部首補正: 月(肉づき)の手動確認（自動判別不可の限界）。
- 旧逆引きモジュールの削除（`readingLookup.ts`/`kanjiapiReading.ts`/`__tests__/readingLookup.test.ts`/`db/seed/name_kanji_allow.json`・未使用）。
- reading指定時の `/api/suggest` は漢字生成でLLMを待つためレイテンシ増。気になる場合はキャッシュ/非同期化。
- ローマ字出力時のかな変換ルール（Phase2で保留・未実装）。
